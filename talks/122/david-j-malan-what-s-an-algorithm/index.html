What's an algorithm?ما هي الخوارزميات؟ In computer science,في علم الحاسب، an algorithm is a set of instructionsالخوارزمية هي مجموعة من التعليمات for solving some problem, step-by-step.لحل مشكلة معينة، خطوة بخطوة. Typically, algorithms are executed by computers,في العادة، تنفذ الخوارزميات باستخدام الحواسيب but we humans have algorithms as well.ولكننا كبشر لدينا خوارزميات أيضًا. For instance, how would you go about countingعلى سبيل المثال، كيف ستقوم بعدّ the number of people in a room?عدد الأشخاص في غرفة؟ Well, if you're like me,حسنا، إن كنت مثلي، you probably point at each person,ستقوم على الأغلب بالتأشير على كل شخص one at a time,كل واحد على حدة and count up from 0:وسوف تعد من الصفر 1, 2, 3, 4 and so forth.1، 2، 3، 4 وهكذا دواليك... Well, that's an algorithm.حسنا، هذه تعتبر خوارزمية. In fact, let's try to express itفي الواقع، نستطيع أن نعبر عنها a bit more formally in pseudocode,بطريقة أكثر رسمية عبر الترميز المستعار، English-like syntaxأي برمجة بجمل تعبيرية that resembles a programming language.تشبه لغة برمجية. Let n equal 0.ليكن n يساوي 0. For each person in room, set n = n + 1.لكل شخص في الغرفة، ليكن n=n+1 . How to interpret this pseudocode?إذن، كيف نترجم هذا الترميز المستعار؟ Well, line 1 declares, so to speak,حسنا، في السطر الأول قمنا بتعريف a variable called nمتغير أسميناه n، and initializes its value to zero.وجعلنا قيمته المبدئية 0. This just means that at the beginning of our algorithm,ما يعنيه هذا هو أنه في بداية خوارزميتنا the thing with which we're countingالشيء الذي نقوم بعدّه has a value of zero.قيمته 0. After all, before we start counting,فقبل أن نبدأ بالعد، we haven't counted anything yet.نكون لم نعد أي شيئ أصلا. Calling this variable n is just a convention.أما تسمية المتغير بالحرف "n" هي مجرد عادة. I could have called it almost anything.وقد كان باستطاعتي تسميته بما أريده. Now, line 2 demarks the start of loop,الآن، السطر الثاني ينوه ببداية "حلقة تكرار"، a sequence of steps that will repeat some number of times.أي خطوات متتالية سوف تعاد لعدد من المرات. So, in our example, the step we're takingإذًا، في مثالنا، الخطوة الواحدة التي نأخذها is counting people in the room.هي عدّ عدد الأشخاص في الغرفة. Beneath line 2 is line 3,تحت السطر الثاني لدينا السطر الثالث which describes exactly how we'll go about counting.والذي يصف تماما كيف سنقوم بعدّنا. The indentation implies that it's line 3المسافة تبين أن السطر الثالث that will repeat.هو الذي سوف يعاد. So, what the pseudocode is sayingإذًا، ما يقوله الترميز المستعار is that after starting at zero,هو أنه بعد البدء من الصفر، for each person in the room,لكل شخص في الغرفة، we'll increase n by 1.سوف نزيد n بمقدار 1. Now, is this algorithm correct?والآن، هل هذه الخوارزمية صحيحة؟ Well, let's bang on it a bit.حسنًا، دعنا نجربها. Does it work if there are 2 people in the room?هل ستعمل إن كان هناك شخصان في الغرفة؟ Let's see.فلنر. In line 1, we initialize n to zero.في السطر الأول، نعيد قيمة n للعدد 0. For each of these two people,لكل من هذين الشخصين، we then increment n by 1.نزيد قيمة n بمقدار 1. So, in the first trip through the loop,إذًا، في أول دورة في حلقة التكرار، we update n from zero to 1,نجدد قيمة n من 0 إلى 1، on the second trip through that same loop,وفي ثاني دورة حول نفس الحلقة، we update n from 1 to 2.نجدد قيمة n من 1 إلى 2. And so, by this algorithm's end, n is 2,وإذًا، عند نهاية الخوارزمية، تكون قيمة n تساوي 2، which indeed matches the number of people in the room.والذي يساوي بالفعل عدد الناس في الغرفة. So far, so good.حسنا، الوضع جيد حتى الآن. How about a corner case, though?ولكن ماذا عن حالة شاذة؟ Suppose that there are zero people in the room,افرض أنه كان هناك 0 من الأشخاص في الغرفة، besides me, who's doing the counting.من غيري، وأنا أقوم بالعد. In line 1, we again initialize n to zero.في السطر الأول، نعيد قيمة n إلى 0. This time, though, line 3 doesn't execute at allولكن في هذه الحالة السطر الثالث لا ينفذ على الإطلاق since there isn't a person in the room,وذلك لأنه لا يوجد شخص في الغرفة أصلًا، and so, n remains zero,ولذلك تبقى قيمة n تساوي ٠، which indeed matches the number of people in the room.وهذا بالفعل عدد الأشخاص في الغرفة. Pretty simple, right?الأمر بسيط، صح؟ But counting people one a time is pretty inefficient, too, no?ولكن عدّ الأشخاص واحدا تلو الآخر عملية بطيئة، أليس كذلك؟ Surely, we can do better!بالتأكيد نستطيع فعل أفضل من ذلك. Why not count two people at a time?لم لا نعد كل شخصين في مرة واحدة؟ Instead of counting 1, 2, 3, 4, 5, 6, 7, 8, and so forth,بدل من العد 1، 2، 3، 4، 5، 6، 7، 8 وهكذا دواليك .. why not countلم لا نعد 2, 4, 6, 8, and so on?2، 4، 6، 8 ... إلخ؟ It even sounds faster, and it surely is.تبدو هذه العملية وكأنها أسرع، وبالتأكيد هي كذلك. Let's express this optimization in pseudocode.فلنعبر عن هذا التسريع باستخدام الترميز المستعار. Let n equal zero.ليكن n يساوي 0. For each pair of people in room,لكل زوج من الأشخاص في الغرفة set n = n + 2.ليكن n=n+2. Pretty simple change, right?تغيير بسيط، أليس كذلك؟ Rather than count people one at a time,بدل من عدّ الناس واحدًا على حدة، we instead count them two at a time.نقوم بعدهم اثنين اثنين. This algorithm's thus twice as fast as the last.وتكون إذًا خوارزميتنا ضعف سرعة الخوارزمية السابقة. But is it correct?ولكن هل هي صحيحة؟ Let's see.فلنر. Does it work if there are 2 people in the room?هل تعمل إن كان هناك شخصان في الغرفة؟ In line 1, we initialize n to zero.في السطر الأول، نعين قيمة n إلى 0. For that one pair of people, we then increment n by 2.للزوج الوحيد في الغرفة، نزيد قيمة n بمقدار 2. And so, by this algorithm's end, n is 2,وعند نهاية الخوارزمية، تكون قيمة n تساوي 2، which indeed matches the number of people in the room.والذي يساوي بالفعل عدد الأشخاص في الغرفة. Suppose next that there are zero people in the room.فلنفرض الآن بأنه هناك 0 من الأشخاص في الغرفة. In line 1, we initialize n to zero.في السطر الأول، نعيد قيمة n إلى صفر. As before, line 3 doesn't execute at allكما في السابق، السطر الثالث لا ينفذ على الإطلاق since there aren't any pairs of people in the room,بما أنه لا توجد أية أزواج في الغرفة، and so, n remains zero,وبالتالي تبقى قيمة n تساوي 0 which indeed matches the number of people in the room.وهو بالفعل عدد الأشخاص في الغرفة. But what if there are 3 people in the room?ولكن ماذا يحصل إن كان هناك 3 أشخاص؟ How does this algorithm fair?كيف ستؤدي خوارزميتنا؟ Let's see.فلنر. In line 1, we initialize n to zero.في السطر الأول، نعيد قيمة n إلى 0. For a pair of those people,لزوج واحد من هؤلاء الأشخاص، we then increment n by 2,نزيد قيمة n بمقدار 2. but then what?ولكن ماذا الآن؟ There isn't another full pair of people in the room,لا يوجد زوج آخر من الأشخاص في الغرفة، so line 2 no longer applies.وبالتالي لن يطبق السطر الثاني. And so, by this algorithm's end,وبالتالي، عند نهاية الخوارزمية n is still 2, which isn't correct.تكون قيمة n لا تزال 2، وهذا خاطئ، Indeed this algorithm is said to be buggyنقول عن هذه الخوارزمية أنه مشوبة because it has a mistake.لأنها تحتوي على خطأ. Let's redress with some new pseudocode.فلنعدلها بترميز مستعار إضافي. Let n equal zero.ليكن n يساوي 0. For each pair of people in room,لكل زوج من الأشخاص في الغرفة، set n = n + 2.ليكن n=n+2. If 1 person remains unpaired,إذا بقي شخص واحد منفردا، set n = n + 1.ليكن n=n+1. To solve this particular problem,لحل هذه المشكلة we've introduced in line 4 a condition,قمنا في السطر الرابع بإضافة شرط، otherwise known as a branch,أو ما يسمى بـ"فرع" that only executes if there is one personوالذي لا ينفذ إلا إن كان هناك شخص واحد we could not pair with another.لم نستطع وضعه في زوج مع شخص آخر. So now, whether there's 1 or 3الآن، سواء كان هناك شخص أو ثلاثة or any odd number of people in the room,أو أي عدد فردي من الأشخاص في الغرفة، this algorithm will now count them.ستقوم الخوارزمية بعدّهم. Can we do even better?هل نستطيع أن نكوّن خوارزمية أفضل؟ Well, we could count in 3's or 4's or even 5's and 10's,حسنا نستطيع أن نعد كل 3، أو كل 4، أو حتى كل 5 و 10 but beyond that it's going to getولكن قد تتعقد الأمور a little bit difficult to point.إذا ذهبنا أبعد من ذلك.
