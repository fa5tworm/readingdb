Qu'est ce qu'un algorithme ?What's an algorithm? En informatique,In computer science, un algorithme est un ensemble d'instructionsan algorithm is a set of instructions pour résoudre un problème étape après étape.for solving some problem, step-by-step. Généralement, les algorithmes sont exécutés par des ordinateursTypically, algorithms are executed by computers, mais les humains ont aussi des algorithmes.but we humans have algorithms as well. Par exemple, comment feriez-vousFor instance, how would you go about counting pour compter le nombre de personnes dans une pièce ?the number of people in a room? Eh bien, si vous êtes comme moi,Well, if you're like me, vous désignez probablement chaque personne,you probably point at each person, une à la fois,one at a time, et commencez à compter à partir de 0 :and count up from 0: 1, 2, 3, 4 et ainsi de suite.1, 2, 3, 4 and so forth. Eh bien ça, c'est un algorithme.Well, that's an algorithm. En fait, essayons de l'exprimerIn fact, let's try to express it de manière un peu plus formelle, en pseudocodea bit more formally in pseudocode, une syntaxe presque françaiseEnglish-like syntax qui ressemble à un langage de programmationthat resembles a programming language. Eh bien, la ligne 1 déclare, pour ainsi dire,Well, line 1 declares, so to speak, une variable appelée na variable called n et initialise sa valeur à zéro.and initializes its value to zero. Ça veut dire qu'au début de notre algorithmeThis just means that at the beginning of our algorithm, la chose avec laquelle nous comptonsthe thing with which we're counting a une valeur de zéro.has a value of zero. Après tout, avant d'avoir commencé à compter,After all, before we start counting, on avait encore rien compté .we haven't counted anything yet. Appeler cette variable n est simplement une convention.Calling this variable n is just a convention. J'aurais pu l'appeler presque n'importe comment.I could have called it almost anything. Maintenant, la ligne 2 marque le début de la boucle,Now, line 2 demarks the start of loop, une séquence d'étapes qui se répète un certain nombre de fois.a sequence of steps that will repeat some number of times. Donc, dans notre exemple, l'étape que nous prenonsSo, in our example, the step we're taking c'est le comptage des personnes dans la salle.is counting people in the room. Après la ligne 2 vient la ligne 3,Beneath line 2 is line 3, qui décrit exactement comment on va procéder.which describes exactly how we'll go about counting. L'indentation sous-entend que c'est la ligne 3The indentation implies that it's line 3 qui va se répéter.that will repeat. Donc, ce que dit le pseudocode,So, what the pseudocode is saying c'est qu'après avoir commencé à zéro,is that after starting at zero, pour chaque personne dans la pièce,for each person in the room, nous allons augmenter n de 1.we'll increase n by 1. Alors, cet algorithme est-il correct ?Now, is this algorithm correct? Eh bien, jetons un coup d’œil là-dessus.Well, let's bang on it a bit. À la ligne 1, nous initialisons n à zéro.In line 1, we initialize n to zero. Pour chacune de ces deux personnes,For each of these two people, nous incrémentons n par 1.we then increment n by 1. Dans le premier parcours à travers la boucle,So, in the first trip through the loop, nous mettons à jour n de zéro à 1,we update n from zero to 1, lors du second parcours à travers cette même boucle,on the second trip through that same loop, nous mettons à jour n de 1 à 2.we update n from 1 to 2. Et donc, à la fin de cet algorithme, n vaut 2,And so, by this algorithm's end, n is 2, ce qui correspond au nombre de personnes dans la salle.which indeed matches the number of people in the room. Pour l'instant ça va.So far, so good. Cependant, si on poussait le raisonnement plus loin ?How about a corner case, though? Assez simple, non ?Pretty simple, right? Mais compter les personnes une à une est plutôt inefficace, non ?But counting people one a time is pretty inefficient, too, no? Plutôt que de compter les gens un par un,Rather than count people one at a time, nous les comptons deux par deux.we instead count them two at a time. Mais que se passe-t-il s'il y a 3 personnes dans la salle ?But what if there are 3 people in the room? Comment s’exécute cet algorithme ?How does this algorithm fair? Nous allons voir.Let's see. À la ligne 1, nous initialisons n à zéro.In line 1, we initialize n to zero. Pour une paire de ces personnes,For a pair of those people, nous incrémentons n par 2,we then increment n by 2, mais alors quoi ?but then what?
